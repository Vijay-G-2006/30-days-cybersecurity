## Technical Log

### Part 1 — Authentication brute force concepts

**Step 1 — Define online brute-force attacks**

Online brute-force attacks attempt many username/password combinations directly against a live service. The main types are:
- **Password brute force** — try a large number of candidate passwords for a single account
- **Password spraying** — try a small number of common passwords across many accounts
- **Credential stuffing** — use leaked username/password pairs from one breach against a different service

All of these attacks must balance speed with stealth to avoid lockouts and detection.

**Step 2 — Why Hydra and similar tools matter**

Hydra and its peers are valuable because they support many protocols and let the attacker automate the workflow:
- `Hydra` — best known for its huge number of service modules
- `Medusa` — optimized for parallel authentication attempts
- `Ncrack` — modern style, built by the Nmap team for high-speed network cracking
- `Patator` — highly flexible, with customizable modules and throttling

### Part 2 — Lab setup and target discovery

**Step 3 — Confirm the target service list**

I used Metasploitable 2 on a host-only network. First, I verified reachable services with Nmap:

```bash
nmap -sV -p 22,21,80,3306,139,445 192.0.2.10
```

The scan showed SSH, FTP, HTTP, MySQL, and SMB services. The most useful next step was identifying login services that support online authentication.

**Step 4 — Use weak credentials in a safe lab**

Since this is a learning exercise, I targeted known weak credentials such as `msfadmin/msfadmin` and a small custom wordlist of common passwords.

### Part 3 — Using Hydra for SSH, FTP, and HTTP

**Step 5 — SSH brute force with Hydra**

Hydra can attack SSH directly with a username and password list:

```bash
hydra -l msfadmin -P ~/wordlists/common-passwords.txt ssh://192.0.2.10
```

This command uses:
- `-l msfadmin` to specify the username
- `-P` to load a password file
- `ssh://<target>` to specify the service and host

**Step 6 — FTP brute force with Hydra**

FTP is another common service for password attacks:

```bash
hydra -L ~/wordlists/usernames.txt -P ~/wordlists/common-passwords.txt ftp://192.0.2.10
```

This tests combinations from a username list and a password list.

**Step 7 — HTTP form attack with Hydra**

HTTP form brute force is more complex because Hydra must know the form fields and failure message.

```bash
hydra -L ~/wordlists/usernames.txt -P ~/wordlists/common-passwords.txt 192.0.2.10 http-post-form "/login.php:username=^USER^&password=^PASS^:F=Login failed"
```

This instructs Hydra to replace `^USER^` and `^PASS^` in the POST body, and to treat responses containing `Login failed` as unsuccessful attempts.

### Part 4 — Comparing other tools

**Step 8 — Medusa for parallel login testing**

Medusa is similar to Hydra but is designed for parallelism and flexibility:

```bash
medusa -h 192.0.2.10 -U ~/wordlists/usernames.txt -P ~/wordlists/common-passwords.txt -M ssh
```

Medusa also supports modules such as `ftp`, `http`, and `telnet`.

**Step 9 — Ncrack for modern network cracking**

Ncrack is built for speed and supports SSH, FTP, and HTTP:

```bash
ncrack -p ssh,ftp 192.0.2.10
```

Ncrack can also reuse Nmap XML output and integrate with existing discovery workflows.

**Step 10 — Patator for custom workflows**

Patator is extremely flexible and can be used for HTTP, SSH, FTP, and many more protocols. A sample command looks like:

```bash
patator ssh_login host=192.0.2.10 user=FILE0 password=FILE1 0=~/wordlists/usernames.txt 1=~/wordlists/common-passwords.txt
```

Patator's ability to throttle requests and define custom success/failure conditions makes it useful for stealthier attacks.

### Part 5 — Defenses and practical observations

**Step 11 — Observe lockout and detection behavior**

In a real environment, an account lockout policy or an intrusion detection system can stop these attacks. In the lab, I observed the following:
- SSH often delayed or disconnected after too many failed attempts
- FTP returned `530 Login incorrect` consistently
- HTTP form attacks required the correct failure string, otherwise Hydra could not tell success from failure

**Step 12 — Rate limiting and account lockout**

I tested how an SSH service reacts to repeated failures and confirmed that slowing the attack with a `-t 4` option in Hydra reduces the chance of triggering basic rate limits.

### Part 6 — Workflow notes and tool selection

Hydra is the go-to tool for broad protocol coverage and quick setup. Medusa is a good alternative when you need parallel control. Ncrack is better for network-focused, high-speed cracking, and Patator is the best choice when you need custom success conditions or slow throttling.

## Challenge Encountered

> **Error / Obstacle:** Hydra initially reported a false negative on the HTTP login form because the failure string was wrong.

**Root cause:** The form returned a different error message than the one configured in the command.

**Resolution:**
1. Loaded the login page in the browser and inspected the exact HTML response.
2. Updated the Hydra command to use the correct failure expression.
3. Re-ran the attack and confirmed valid credentials were discovered.

> **Key realization:** HTTP brute force only works when the attacker correctly models the web form and failure behavior. The same command works for SSH/FTP but not for web forms without careful configuration.

## Key Takeaways

- **Hydra is highly versatile.** It supports dozens of protocols and is the first tool to reach for on online login attacks.
- **Not all protocols are equal.** SSH/FTP are straightforward; HTTP forms require exact request templates and failure detection.
- **Tool choice matters.** Medusa excels at parallel jobs, Ncrack is fast and network-focused, and Patator is best for custom workflows.
- **Defense practices matter.** Rate limiting, account lockout, and response normalization can stop these attacks before credentials are exhausted.
- **Local labs are safe.** A host-only Metasploitable 2 instance is the correct environment for testing brute-force tools.
