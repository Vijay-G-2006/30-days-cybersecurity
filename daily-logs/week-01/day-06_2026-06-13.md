## Overview

## Key Concepts

*   **One-Way Function:** Hashing is a one-way process. Unlike encryption, a hash cannot technically be "decrypted" back to its original plaintext. Instead, we "crack" it by hashing potential inputs until we find a match.
*   **Brute-Force Attack:** The process of guessing the original input by generating many potential inputs (like sequential numbers or words from a dictionary), hashing them, and comparing the resulting output to the target hash.
*   **Python `hashlib`:** A built-in Python library that provides a common interface to many different secure hash and message digest algorithms (e.g., SHA-256, SHA-512, MD5).
*   **Weak Inputs:** This exercise demonstrates why simple passwords (like short, numeric PINs) are highly insecure. An automated script can guess millions of numbers in just a few seconds.

## Practical Exercise: Automating Hash Matching

Below is a refined, well-commented Python script based on today's learning. This script attempts to find the original numeric input for a given SHA-512 hash by brute-forcing numbers from 1 to 9,999,999.

*(Note: The original script printed every single attempt to the console, which significantly slows down execution. This refined version runs silently until a match is found to maximize speed.)*

### The Code

```python
import hashlib
import time

def brute_force_numeric_hash(target_hash: str, max_range: int = 9999999) -> None:
    """
    A basic script demonstrating how weak passwords (like short numbers)
    can be brute-forced by hashing predictable inputs and comparing them.
    """
    print(f"[*] Starting brute-force attempt for target hash: {target_hash[:10]}...")
    start_time = time.time()
    
    for number in range(1, max_range + 1):
        # 1. Convert the number to a string
        # 2. Encode it to bytes (hashlib requires bytes-like objects)
        input_bytes = str(number).encode('utf-8')
        
        # 3. Calculate the SHA-512 hash and get the hexadecimal representation
        resultant_hash = hashlib.sha512(input_bytes).hexdigest()
        
        # 4. Check if we have a match with our target
        if resultant_hash == target_hash:
            elapsed_time = time.time() - start_time
            print(f"\n[+] Success! A match was found.")
            print(f"[+] Matching input: {number}")
            print(f"[+] Time taken: {elapsed_time:.2f} seconds")
            return
            
    print("\n[-] No match found in the given range.")

if __name__ == "__main__":
    # Example SHA-512 hash 
    # (The hash for a simple numeric input)
    example_hash = '18518656fba871522838d32029e9374606125bb637f13a754759c2200af8e76e5e1be01db7f7c'
    
    # Run the brute-force function
    brute_force_numeric_hash(example_hash)
```

### Code Explanation

1.  **`import hashlib`**: Imports the necessary Python library for generating cryptographic hashes.
2.  **`str(number).encode('utf-8')`**: Hash functions operate on raw bytes, not strings or integers. We must first convert the number to a string, and then encode that string into utf-8 bytes.
3.  **`hashlib.sha512(...)`**: Creates a new SHA-512 hash object.
4.  **`.hexdigest()`**: Returns the digest (the resulting hash) as a string of hexadecimal digits, which is the standard way to read and compare hashes.

## Takeaways and OPSEC Applications

*   **Avoid Predictable Patterns:** Never use simple numbers, dates, or predictable patterns as passwords or keys. They can be cracked almost instantly with modern processing power.
*   **Use Salting:** In real-world applications (like storing user passwords in a database), hashes must be "salted" (adding a unique, random string of data to the input before hashing) to prevent pre-computed dictionary attacks and rainbow tables.
*   **Use Strong Algorithms:** While SHA-512 is a strong, collision-resistant hash, it is designed to be fast. For storing passwords, slow, key-derivation algorithms (like Argon2, bcrypt, or PBKDF2) should be used to make brute-forcing computationally expensive and economically unviable for attackers.
